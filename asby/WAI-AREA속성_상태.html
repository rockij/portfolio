WAI-ARIA 동영상 

                    -요소(Element)가 기본적으로 갖고 있는 특성이나 상황
                    -속성과 상태는 "aria-*" 접두어를 가진다.
                    -상태는 요소의 상황을 나태내므로 애플리케이션이 실행중에 자주 바뀌는 반면, 속성은 상대적으로 바뀌는 경우가 드물다.

필수 항목 (속성)

스크린리더 사용자는 해당 입력 서식이 필수 항목임을 전달 받게됨
<input type="checkbox" aria-required="true">

추가 설명 (속성)

                    예를들어 아래 입력박스에서 최소 10글자 이상 입력해야한다는 부가정보를 사용자에게 알리고자 할 때 추가 설명이 기재된 요소에 고유의 id를 지정 한 후 입력서식에 aria-describedby 속성의 값으로 입력 이 경우 스크린리더 사용자의 초점이 입력 서식에 위치하는 순간 추가 설명을 함께 읽어주기 때문에 입력 값을 좀 더 명확히 이해할 수 있게 된다.
<input type="text" aria-describedby="reference"> <div id="reference">추가설명</div>

그룹 레이블 명명 (속성)

                    -다양한 요소의 레이블을 지정하는 용도로 사용 가능
                    -긴 설명보다는 명사 등 간단한 이름으로 정의하여 해당 요소의 성격이 무엇인지 이해를 돕도록 한다.
<div role="group" aria-label="레이블">

확장되어 있는 상태의 탭패널 (상태)

                    값으로 true를 지정하면 펼쳐진 상태를 의미하며 false가 지정되면 접혀진 상태 정보를 사용자에게 전달한다.
<div role="tabpanel" aria-expanded="true">

오류가 발생한 상태의 입력상자 (상태)

                    -입력박스의 값이 유효한지 아닌지 오류 발생 상태 정보를 제공할때
                    -true의 경우 오류가 발생한 것을 의미
<input type="text" aria-invalid="true">

선택된 상태의 토글버튼

                    -토글버튼의 상태정보를 나타날때 사용
                    -true의 경우 버튼이 눌러진 상태를 의미
<button type="button" aria-pressed="true">
